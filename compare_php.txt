 <h1>PHP 型の比較表</h1>

 <p class="para">
  下記の表はPHPの<a href="language.types.php" class="link">型</a>と
  <a href="language.operators.comparison.php" class="link">比較演算子</a>
  の振る舞いについて、緩やかな場合と厳密な場合の両方について
  例を示しています。この付録はマニュアルの
  <a href="language.types.type-juggling.php" class="link">型の相互変換</a>
  にも関連しています。種々のユーザーコメントと
  <a href="http://www.blueshoes.org/en/developer/php_cheat_sheet/" class="link external">&raquo;&nbsp;BlueShoes</a>の働きの
  おかげです。
 </p>
 <p class="para">
  この表を活用する前に、型とその意味について理解しておく必要があります。
  例えば、<em>&quot;42&quot;</em>は<span class="type"><span class="type 文字列">文字列</span></span>ですが
  <em>42</em>は<span class="type"><span class="type 整数">整数</span></span>です。
  <strong><code>FALSE</code></strong>は<span class="type"><a href="language.types.boolean.php" class="type boolean">boolean</a></span>ですが<em>&quot;false&quot;</em>は
  <span class="type"><span class="type 文字列">文字列</span></span>です。
 </p>
 <blockquote class="note"><p><strong class="note">注意</strong>: 
  <p class="para">
   HTMLフォームは整数、浮動小数点数、booleanを渡してはくれず、
   文字列を渡します。文字が数値であるかどうか確認するには、
   <span class="function"><a href="function.is-numeric.php" class="function">is_numeric()</a></span>を使うとよいでしょう。
  </p>
 </p></blockquote>
 <blockquote class="note"><p><strong class="note">注意</strong>: 
  <p class="para">
   <var class="varname"><var class="varname">$x</var></var>が定義されていない状態で単に
   <em>if ($x)</em>としてしまうと<strong><code>E_NOTICE</code></strong>
   レベルのエラーが発行てしまいます。代わりに、<span class="function"><a href="function.empty.php" class="function">empty()</a></span>や
   <span class="function"><a href="function.isset.php" class="function">isset()</a></span>を使うかあるいは変数を初期化するように
   してください。
  </p>
 </p></blockquote>
 <blockquote class="note"><p><strong class="note">注意</strong>: 
  <p class="para">
   数値演算の結果が、定数 <strong><code>NAN</code></strong> で表される値になることがあります。
   この値を、自分自身を含む他の値と比較すると、緩い比較であっても厳しい比較であっても結果は
   <strong><code>FALSE</code></strong> となります
   (つまり <em>NAN != NAN</em> であり <em>NAN !== NAN</em> であるということです)。
   結果が <strong><code>NAN</code></strong> になる演算の例には
   <em>sqrt(-1)</em> や <em>asin(2)</em> そして
   <em>asinh(0)</em> があります。
  </p>
 </p></blockquote>

<div class="phpcode"><code><span class="html">
In some languages, a boolean is promoted to an integer (with a value of 1 or -1, typically) if used in an expression with an integer. I found that PHP has it both ways:<br /><br />If you add a boolean with a value of true to an integer with a value of 3, the result will be 4 (because the boolean is cast as an integer).<br /><br />On the other hand, if you test a boolean with a value of true for equality with an integer with a value of three, the result will be true (because the integer is cast as a boolean).<br /><br />Surprisingly, at first glance, if you use either &lt; or &gt; as the comparison operator the result is always false (again, because the integer as cast as a boolean, and true is neither greater nor less than true).</span>
</code></div>

PHP's loose comparisons can be a huge convenience when used properly! It's extremely helpful to just remember the following are always FALSE:<br /><br />null, false, "", 0, "0", array()<br /><br />If your application never depends on a particular "empty/false/null/0/not set" value type, you won't have to worry about 99% of the other weird cases listed here. You won't need empty() or isset(). And ALL variable types will always work as expected for statements like:<br /><br />if(boolean &amp;&amp; !string){...}<br />if(array){...}<br />if(!null || int){...}<br /><br />Consider the same when working with your database values.</span>
